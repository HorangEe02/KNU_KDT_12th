## 1. 프로젝트 전체 구조

```
{분야명}_analysis/
├── config/                          ← 01번 가이드
│   ├── __init__.py
│   └── settings.py                  ★ 핵심 설정 파일 (이것만 바꾸면 80% 완료)
├── crawlers/                        ← 02번 가이드
│   ├── __init__.py
│   ├── naver_api_crawler.py         네이버 블로그/뉴스 (OpenAPI)
│   ├── saramin_crawler.py           사람인 채용공고 (requests + BS4)
│   ├── wanted_crawler.py            원티드 채용공고 (REST API v4)
│   └── jobplanet_crawler.py         잡플래닛 리뷰/면접 (Selenium)
├── processors/                      ← 03번 가이드
│   ├── __init__.py
│   ├── data_cleaner.py              데이터 전처리
│   ├── text_analyzer.py             텍스트/형태소 분석
│   └── salary_analyzer.py           연봉 분석
├── data/                            ← 04번 가이드
│   ├── raw/                         크롤링 원본 CSV/JSON
│   └── processed/                   전처리 완료 CSV
├── visualizations/                  ← 05번 가이드
│   ├── __init__.py
│   ├── wordcloud_viz.py             워드클라우드 7종
│   ├── chart_viz.py                 기본 차트 11종
│   ├── advanced_viz.py              고급 차트 10종
│   ├── interactive_viz.py           Plotly 인터랙티브 8종
│   ├── dashboard.py                 종합 대시보드
│   ├── content_trend_viz.py         콘텐츠 트렌드 5종
│   ├── cross_analysis_viz.py        교차 분석 4종
│   ├── jobplanet_deep_viz.py        잡플래닛 심층 6종
│   └── wanted_viz.py                원티드 전용 7종
├── outputs/                         ← 06번 가이드
│   ├── charts/                      시각화 결과 (PNG + HTML)
│   └── reports/                     분석 보고서
├── url_list/                        수집 URL 목록
│   └── crawling_urls.csv
├── main.py                          ← 07번 가이드
└── cross.png                        워드클라우드 마스크 이미지
```

## 2. 5단계 파이프라인

```
[Phase 1] 크롤링 ──→ [Phase 2] 전처리 ──→ [Phase 3] 분석 ──→ [Phase 4] 시각화 ──→ [Phase 5] URL 정리
  crawlers/           processors/          processors/         visualizations/       url_list/
  4개 소스             data_cleaner         text_analyzer       9개 모듈              crawling_urls.csv
                                           salary_analyzer     52개 차트
```

| Phase | 입력 | 출력 | 담당 모듈 |
|-------|------|------|----------|
| 1. 크롤링 | 키워드, 기업 ID, API 키 | `data/raw/*.csv` | `crawlers/` |
| 2. 전처리 | `data/raw/*.csv` | `data/processed/*.csv` | `processors/data_cleaner` |
| 3. 분석 | `data/processed/*.csv` | 빈도 CSV, 통계 dict | `processors/text_analyzer, salary_analyzer` |
| 4. 시각화 | 분석 결과 | `outputs/charts/*.png, *.html` | `visualizations/` |
| 5. URL 정리 | 크롤링 URL 로그 | `url_list/crawling_urls.csv` | `main.py` |

## 4. 분야 전환 시 수정 우선순위

```
[1순위] config/settings.py    ← 이것만 바꾸면 80% 완료 (★★★ 가이드 01번)
[2순위] crawlers/ 파일명 접미사 + 원티드 카테고리 ID (★★☆ 가이드 02번)
[3순위] processors/ 도메인 용어 매핑 (★★☆ 가이드 03번)
[4순위] visualizations/ 차트 제목 (★☆☆ 가이드 05번)
[5순위] main.py 파일명 패턴 (★☆☆ 가이드 07번)
```

## 6. 기업 선정 기준 공식

```
= 분야 대표 기업 3~4개 (시장 점유율 상위)
+ 세부 분야별 대표 1~2개 (기술 다양성)
+ 스타트업 3~4개 (혁신/성장성)
+ 중소기업 2~3개 (안정성)
+ 대기업 1~2개 (규모/인프라)
= 총 10~15개
```

## 7. 키워드 설계 공식

### 블로그 키워드 (15~25개)

```
= "{분야명} 취업 준비"        ← 취업 준비 일반 (3~5개)
+ "{분야명} {직무} 취업"
+ "{분야명} {핵심기술} 개발자"  ← 직무 특화 (3~5개)
+ "{분야명} 자격증"            ← 역량 개발 (3~5개)
+ "{분야명} 포트폴리오"
+ "{분야명} 인턴"              ← 경력 단계 (2~3개)
+ "{분야명} 신입 채용"
+ "{분야명} {세부기술}"        ← 기술 키워드 (3~5개)
```

### 뉴스 키워드 (10~15개)

```
= "{분야명} 채용"              ← 채용 현황 (3~4개)
+ "{분야명} 인력"
+ "{분야명} 산업 전망"          ← 산업 전망 (3~4개)
+ "{분야명} 시장"
+ "{분야명} 연봉"              ← 보상/투자 (2~3개)
+ "{분야명} 투자"
+ "{분야명} 규제"              ← 규제/정책 (2~3개)
```
