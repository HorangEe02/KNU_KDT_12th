### 핵심 원칙: `settings.py`만 바꾸면 80%가 완료된다

이 프로젝트의 가장 큰 장점은 **모든 설정이 `config/settings.py`에 집중**되어 있다는 것이다. 분야를 바꿀 때 `settings.py`의 키워드, 대상 기업, 도메인 용어만 수정하면 나머지 모듈은 최소한의 변경으로 재사용할 수 있다.

## 3. STEP 1 - settings.py 설정 변경

`settings.py`는 전체 프로젝트의 **중앙 설정 파일**이다. 이 파일만 제대로 수정하면 나머지 모듈이 자동으로 새 분야에 맞춰 동작한다.

### 3.1 수정 항목 총정리

```python
# ============================================================
# [필수 수정 1] .env 파일 생성 (프로젝트 루트)
# ============================================================
# .env 파일에 API 키와 로그인 정보를 저장 (git 제외됨)
# settings.py에서 os.getenv()로 자동 로드
#
# .env 파일 내용:
# NAVER_CLIENT_ID=YOUR_CLIENT_ID
# NAVER_CLIENT_SECRET=YOUR_CLIENT_SECRET
# JOBPLANET_USER_ID=your_email@example.com
# JOBPLANET_PASSWORD=your_password
#
# settings.py에서는 다음과 같이 참조:
# NAVER_CLIENT_ID = os.getenv("NAVER_CLIENT_ID", "")
# NAVER_CLIENT_SECRET = os.getenv("NAVER_CLIENT_SECRET", "")
# JOBPLANET_USER_ID = os.getenv("JOBPLANET_USER_ID", "")
# JOBPLANET_PASSWORD = os.getenv("JOBPLANET_PASSWORD", "")

# ============================================================
# [필수 수정 3] 대상 기업 리스트
# ============================================================
# 잡플래닛 company_id는 잡플래닛 기업 페이지 URL에서 확인

    "기업명": {
        "company_id": "잡플래닛_기업ID",
        "category": "카테고리",
        "비고": "기업 설명",
    },
    # ... 10~15개 기업 추가
}
```

## 4. STEP 2 - 크롤러 수정

### 4.1 naver_api_crawler.py - 수정 최소

| 수정 항목 | 수정 내용 | 난이도 |
|----------|----------|--------|
| 키워드 | settings.py에서 자동 참조 → **수정 불필요** | - |
| 파일명 접미사 | `_의료AI` → `_{분야명}` 으로 변경 | ★☆☆ |
| 검색 옵션 | `display`, `sort` 파라미터 조정 (필요 시) | ★☆☆ |

```python
# 변경 포인트: run_all() 메서드의 파일명
def run_all(self):
    # 기존: "naver_blog_의료AI", "naver_news_의료AI"
    # 변경: "naver_blog_{분야명}", "naver_news_{분야명}"
    df_blog = self.search_all_keywords(NAVER_BLOG_KEYWORDS, "blog")
    self.save_data(df_blog, "naver_blog_{분야명}")
    # ...
```

**핵심**: 이 크롤러는 네이버 OpenAPI를 사용하므로 분야와 무관하게 **동일한 구조**로 작동한다. settings.py의 키워드만 바꾸면 된다.

### 4.2 saramin_crawler.py - 수정 최소

| 수정 항목 | 수정 내용 | 난이도 |
|----------|----------|--------|
| 키워드 | settings.py에서 자동 참조 → **수정 불필요** | - |
| 기술스택 추출 | settings.py의 TECH_STACK_KEYWORDS 참조 → **수정 불필요** | - |
| 자격증 추출 | settings.py의 CERTIFICATE_KEYWORDS 참조 → **수정 불필요** | - |
| 파일명 | `run_all()` / `run_custom()` 내 저장 파일명 변경 | ★☆☆ |

**핵심**: 사람인 크롤러는 검색 결과 카드(`.item_recruit`)에서 직접 데이터를 추출하는 방식이다. 상세 페이지는 JavaScript 렌더링으로 접근이 제한되므로, 검색 카드 기반 추출을 사용한다. 사람인의 HTML 구조가 변경되지 않는 한 크롤러 로직 자체는 수정 불필요. 키워드와 기술스택 사전만 settings.py에서 교체하면 된다.

### 4.3 wanted_crawler.py - 카테고리 ID 변경 필요

| 수정 항목 | 수정 내용 | 난이도 |
|----------|----------|--------|
| **CATEGORY_ID** | 원티드 카테고리 ID를 분야에 맞게 변경 | ★★☆ |
| **SUBCATEGORY_MAP** | 하위 카테고리 매핑 딕셔너리 재정의 | ★★☆ |
| 파일명 | 저장 파일명 변경 | ★☆☆ |

```python
class WantedCrawler:
    # 기존: 의료/제약/바이오
    # CATEGORY_ID = 515
    # 변경 예시: 금융
    CATEGORY_ID = 507

    # 하위 카테고리 매핑도 재정의
    SUBCATEGORY_MAP = {
        # 원티드 API 응답의 subcategory 번호 → 한글명
        10110: "은행",
        10111: "증권",
        10112: "보험",
        10113: "핀테크",
        # ...
    }
```

**확인 방법**: 원티드 웹사이트에서 원하는 카테고리를 선택한 후 브라우저 개발자 도구 Network 탭에서 API 요청의 `tag_type_ids` 값을 확인한다.

### 4.4 jobplanet_crawler.py - 대상 기업 변경

| 수정 항목 | 수정 내용 | 난이도 |
|----------|----------|--------|
| TARGET_COMPANIES | settings.py에서 기업 목록 변경 → **자동 적용** | - |
| CSS 선택자 | 잡플래닛 HTML 구조 변경 시 수정 필요 (드묾) | ★★★ |
| 파일명 | 저장 파일명 변경 | ★☆☆ |

**핵심**: settings.py의 `TARGET_COMPANIES` 딕셔너리만 바꾸면 자동으로 새 기업들을 크롤링한다. 잡플래닛의 HTML 구조가 동일하므로 크롤링 로직 자체는 수정 불필요.

**기업 ID 찾는 법**:
1. 잡플래닛 접속 → 기업 검색
2. 기업 페이지 URL 확인: `https://www.jobplanet.co.kr/companies/329047/reviews/뷰노`
3. URL의 숫자가 `company_id` (`329047`)
