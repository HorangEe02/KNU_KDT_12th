## 1. 프로젝트 개요

### 1.1 배경 및 목적

의료 AI는 영상의학, 디지털 헬스케어, 신약 개발 등 다양한 의료 분야에서 급속히 성장하고 있다. 그러나 이 분야의 채용 시장에 대한 체계적인 데이터 분석은 부족한 실정이다. 본 프로젝트는 **4개 플랫폼(네이버, 사람인, 원티드, 잡플래닛)**에서 의료 AI 관련 데이터를 자동 수집·전처리·분석·시각화하는 **엔드투엔드 파이프라인**을 구축하여, 의료 AI 취업 시장의 현황과 동향을 정량적으로 파악하는 것을 목적으로 한다.

### 1.2 분석 대상

- **콘텐츠 분석**: 네이버 블로그(532건), 네이버 뉴스(451건)
- **채용 시장 분석**: 사람인(2,078건), 원티드(74건)
- **기업 내부 분석**: 잡플래닛(기업정보 12건, 리뷰 155건, 면접후기 101건)
- **타겟 기업**: 뷰노, 루닛, 딥노이드, 제이엘케이, 셀바스AI, 라인웍스, 메디컬에이아이, 닥터나우, 인피니트헬스케어, 카카오헬스케어, 네이버클라우드(의료), 솔트룩스 (총 12개사)

### 1.3 파이프라인 구조

```
Phase 1: 크롤링 (4개 소스)
    ↓
Phase 2: 전처리 (HTML 제거, 의료용어 정규화, 중복 제거)
    ↓
Phase 3: 텍스트 분석 (형태소 분석, 기술스택/자격증/도메인 빈도)
    ↓
Phase 4: 시각화 (10개 모듈, 70종 차트)
    ↓
Phase 5: URL 목록 정리 (3,262개 고유 URL)
```

## 5. 최종 결론

### 결론 1: Python + AI/ML은 의료 AI 취업의 필수 역량이다

Python(790회, 1위)은 의료 AI 분야의 사실상 표준 언어이며, 딥러닝(598회)과 머신러닝(585회)이 기본 요구사항으로 자리잡았다. 기술스택의 약 62%가 프로그래밍 언어와 AI/ML 프레임워크에 집중되어 있어, **코딩 능력과 AI 모델링 역량이 진입의 최소 조건**임을 확인하였다.

### 결론 2: 의료 도메인 지식이 핵심 차별화 요소이다

기술스택만으로는 차별화가 어렵다. 의료영상(245회), 임상시험(195회), FDA/MFDS 인허가(262회) 등 **의료 도메인 키워드가 기술스택 빈도의 약 13%**를 차지하며, 이는 순수 IT 인력 대비 의료 지식을 갖춘 융합 인력의 경쟁 우위를 시사한다. 특히 방사선사(46.6%)와 임상병리사(19.2%)가 자격증 수요의 66%를 차지하여, **의료 면허 보유자의 AI 역량 확보**가 취업 시장에서 강력한 차별화 요소가 된다.

### 결론 3: 영상의학이 의료 AI의 최대 응용 분야이다

의료 도메인 키워드 분석에서 영상의학이 **53%로 과반 이상**을 차지하였다. 의료영상(245), CT(239), MRI(15), X-ray(14) 등 영상의학 관련 키워드가 압도적이며, 이는 뷰노·루닛·딥노이드 등 국내 주요 의료 AI 스타트업이 대부분 영상의학 분야에서 출발한 산업 구조를 반영한다.

### 결론 4: 정규직 중심의 안정적 채용 구조이나 스타트업 만족도는 낮다

사람인 2,078건 분석 결과 **정규직이 77.7%**로 고용 안정성이 높은 편이다. 그러나 잡플래닛 리뷰 분석에서 **업계 평균 별점이 2.94점**(5점 만점)으로 나타났으며, 특히 스타트업(평균 ~2.7점)의 직원 만족도가 대기업(평균 ~4.3점) 대비 현저히 낮아 **빠른 성장과 근무 환경 간의 트레이드오프**가 존재한다.

### 결론 5: 수도권 집중과 온라인 채용이 채용 시장의 구조적 특성이다

채용 포지션의 대다수가 서울·경기 수도권에 집중되어 있으며, 면접 경로의 **73.2%가 온라인 지원**으로 확인되었다. 이는 의료 AI 기업이 IT 클러스터(강남·판교·구로)에 밀집해 있고, 채용 프로세스가 온라인 플랫폼 중심으로 운영됨을 의미한다.

| **Tier 2 (차별화)** | R/통계분석, 의료영상 지식, 임상시험 이해 | 빈도 4~6위, 도메인 키워드 75% | 의료통계, DICOM 표준, 임상 프로토콜 학습 |
| **Tier 3 (경쟁력)** | AWS/클라우드, Docker/MLOps, NLP | 빈도 8위, 64~71회 | 클라우드 배포, ML 파이프라인 자동화 |

#### 배경별 자격증 전략

- **의료 전공자**: 방사선사/임상병리사 면허가 최강의 경쟁력 (자격증 수요의 66%)
- **비의료 전공자**: ADP(15.1%), 정보처리기사(4.1%), SQLD(4.1%) 순으로 취득 권장
- **공통**: 의료 도메인 지식(영상의학, 인허가, 임상) 학습 병행 필수

#### 지원 전략

- **온라인 채용 플랫폼 활용**: 면접의 73.2%가 온라인 지원 경로 → 사람인·원티드·잡플래닛 프로필 관리 중요
- **기업 선택 기준**: 안정성 → 네이버클라우드(4.87점), 카카오헬스케어(3.67점) / 성장성 → 뷰노, 루닛 (별점은 낮으나 기술력 높음)

### 6.2 교육기관을 위한 제언

- **융합 교육 강화**: 단순 AI 기술 교육이 아닌, 의료영상(DICOM, CT/MRI) + AI + 인허가(FDA/MFDS) 융합 커리큘럼 설계 필요
- **통계 역량 강화**: R(535회, 4위)과 통계분석 도구(SPSS, SAS)가 상위권에 위치 → 의료통계·임상통계 교육 중요성 확인
- **실습 중심 교육**: Python + PyTorch/TensorFlow 기반 의료영상 분석 프로젝트 실습 병행

### 6.3 산업계를 위한 시사점

- **인력 부족의 구조적 원인**: 의료 면허 + AI 기술을 동시에 보유한 인력의 절대적 부족 → 기업 내부 교차 교육(Cross-training) 프로그램 필요
- **직원 만족도 개선**: 업계 평균 별점 2.94점 → 특히 스타트업의 근무 환경, 워라밸, 보상 체계 개선이 인력 유지에 필수적
- **인허가 역량 확보**: SaMD(70), FDA(51), MFDS(14) 관련 키워드가 도메인의 24%를 차지 → 규제 대응 전문인력 확보가 사업화의 핵심

## 7. 한계점

### 7.1 데이터 수집의 한계

| 한계 | 상세 내용 | 영향 |
|------|----------|------|
| **API 제공 범위** | 네이버 OpenAPI는 제목과 요약(description)만 제공, 본문 전체 텍스트는 미포함 | 블로그/뉴스 텍스트 분석의 깊이 제한 |
| **사람인 상세 페이지 미수집** | JavaScript 렌더링이 필요한 상세 페이지를 크롤링하지 못하고 검색 결과 카드 정보만 수집 | 자격요건·우대사항 등 상세 정보 누락, 기술스택 추출이 키워드 매칭에 의존 |
| **원티드 데이터 규모** | 의료/제약/바이오 카테고리 특성상 74건으로 적은 표본 | 통계적 대표성 제한 |
| **잡플래닛 유료 콘텐츠** | 리뷰 본문(장점/단점), 상세 연봉 정보는 유료 멤버십 뒤에 있어 수집 불가 | 정성적 분석(직원 경험, 연봉 비교) 불가 |
| **잡플래닛 기업 한정** | 사전 정의된 12개 기업만 타겟 | 신생 기업이나 비주류 기업 데이터 누락 |

### 7.2 분석 방법론의 한계

| 한계 | 상세 내용 | 영향 |
|------|----------|------|
| **키워드 매칭 방식** | 기술스택·자격증 추출이 사전 정의된 키워드 리스트에 의존 | 새로운 기술이나 변형 표기를 놓칠 수 있음 |
| **형태소 분석기 한계** | KoNLPy Okt는 의료 전문용어 분석에 최적화되어 있지 않음 | "의료영상AI", "디지털헬스케어" 등 복합어 분리 시 의미 손실 가능 |
| **시점 한정** | 2026년 2월 기준 단일 시점 크롤링 | 시간에 따른 채용 트렌드 변화 추적 불가 (월별 추이는 콘텐츠 발행일 기준으로 일부 가능) |
| **연봉 데이터 부재** | 사람인 검색 카드에 연봉 미기재, 원티드의 연봉범위는 실제 경력범위, 잡플래닛 연봉 유료 | 보상 수준 비교 분석 불가 |
| **인과관계 추론 불가** | 빈도 기반 분석으로 상관관계만 파악 가능 | 예: 기업 별점과 면접 난이도 간 상관관계는 확인 가능하나 인과관계는 확인 불가 |

### 7.3 시각화의 한계

- **파이/도넛 차트**: 카테고리가 많을 경우 소수 항목의 라벨 겹침 문제 발생 (3건 수정 완료: saramin_05, wanted_03, jobplanet_03). 전체 파이/도넛 차트의 글씨 크기를 PPT 삽입 시에도 가독성이 확보되도록 라벨 14pt, 퍼센트 13~14pt, 범례 14pt, 제목 18pt로 상향 조정 완료
- **한글 폰트 의존성**: macOS 환경에서 Apple SD Gothic Neo 등 시스템 폰트에 의존하며, 다른 OS에서는 폰트 설정 수정 필요
- **정적 차트 한계**: PNG 차트는 상호작용 불가 → Plotly HTML로 보완했으나, 모든 차트에 적용하지는 못함
