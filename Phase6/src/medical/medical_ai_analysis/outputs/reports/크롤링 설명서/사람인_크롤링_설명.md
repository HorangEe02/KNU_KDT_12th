# 사람인 채용공고 크롤링 (2,078건)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **데이터 소스** | 사람인 (saramin.co.kr) |
| **수집 건수** | 2,078건 |
| **수집 목적** | 의료 AI 분야 채용 시장 현황 파악 (직무, 경력, 학력, 지역, 기술스택 등) |
| **수집 일시** | 2026-02-21 |
| **크롤링 방식** | requests + BeautifulSoup (검색 결과 카드 파싱) |

## 2. 수집 방식

### 2.1 사용 기술
- **HTTP 클라이언트**: `requests.Session` (쿠키/세션 유지)
- **HTML 파서**: `BeautifulSoup` (검색 결과 카드 DOM 파싱)
- **구현 파일**: `crawlers/saramin_crawler.py` > `SaraminCrawler` 클래스

### 2.2 수집 흐름

```
키워드 리스트 (12개)
    ↓
키워드별 검색 페이지 순회 (최대 5페이지/키워드)
    ↓
검색 결과 카드(.item_recruit) 파싱
    ↓
공고URL 기반 중복 제거
    ↓
기술스택/자격증 키워드 추출
    ↓
CSV 저장 (data/raw/)
```

1. `config/settings.py`에 정의된 12개 검색 키워드를 순회
2. 키워드별 최대 5페이지까지 검색 (페이지당 약 40개 공고)
3. 검색 결과 카드(`.item_recruit`)에서 직접 데이터 추출
4. `공고URL` 기준 중복 제거
5. 제목+직무분야 텍스트에서 기술스택/자격증 키워드 자동 추출
6. 페이지 간 2~4초 랜덤 딜레이 적용

### 2.4 파싱 대상 요소

검색 결과 카드(`.item_recruit`)에서 추출하는 항목:

| CSS 셀렉터 | 추출 데이터 |
|-------------|-------------|
| `.job_tit a` | 공고 제목 + URL |
| `.corp_name a` | 회사명 |
| `.job_condition > span` | 경력조건, 학력조건, 근무지역, 고용형태 |
| `.job_sector a` | 직무분야 |
| `.job_date .date` | 마감일 |
| `.job_day` | 등록일 |
| `.area_badge .badge` | 배지 (투자유치, 신입환영 등) |

## 6. 데이터 특성

- **크롤링 범위**: 검색 결과 카드(목록 페이지)에서 추출하며, 상세 페이지는 JavaScript 렌더링이 필요하여 접근하지 않음
- **재시도 로직**: HTTP 오류 시 최대 3회 재시도 (`max_retries=3`)
- **안티봇 대응**: User-Agent 설정, Referer 헤더, 랜덤 딜레이(2~4초)
- **데이터 한계**:
  - 검색 카드에서 추출 가능한 정보만 수집 (상세 자격요건, 우대사항 등은 미포함)
  - 기술스택은 공고 제목/직무분야 텍스트에서 키워드 매칭으로 추출하므로 누락 가능
