# 네이버 뉴스 크롤링 (451건)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **데이터 소스** | Naver OpenAPI (뉴스 검색) |
| **수집 건수** | 451건 |
| **수집 목적** | 의료 AI 산업 트렌드, 채용 현황, 시장 동향 파악 |
| **수집 일시** | 2026-02-20 |
| **API 엔드포인트** | `https://openapi.naver.com/v1/search/news.json` |

## 2. 수집 방식

### 2.1 사용 기술
- **API**: Naver OpenAPI (검색 > 뉴스)
- **인증**: `X-Naver-Client-Id` / `X-Naver-Client-Secret` 헤더 인증
- **구현 파일**: `crawlers/naver_api_crawler.py` > `NaverAPICrawler.search_news()`

### 2.2 수집 흐름

```
키워드 리스트 (15개)
    ↓
키워드별 API 호출 (최대 30건/키워드, 최신순 정렬)
    ↓
URL 기반 중복 제거
    ↓
CSV / JSON 저장 (data/raw/)
```

1. 설정 파일(`config/settings.py`)에 정의된 15개 뉴스 검색 키워드를 순회
2. 키워드당 최대 30건씩 수집 (정렬: `sort="date"` 최신순)
3. `원본링크` 기준으로 중복 URL 실시간 제거
4. API 호출 간 0.1초 딜레이 적용
5. 결과를 CSV + JSON 형식으로 `data/raw/`에 저장

### 2.3 검색 키워드 (15개)

| 분류 | 키워드 |
|------|--------|
| 채용/인력 | 의료 AI 채용, 헬스케어 AI 인력, 의료 AI 스타트업 채용, 디지털 헬스케어 채용 |
| 산업 전망 | 의료 AI 산업 전망, 의료 AI 투자, 헬스케어 AI 시장 |
| 인력 수요 | 의료 AI 인력 수요, 의료 AI 개발자 부족, 의료 AI 기술 인재 |
| 연봉/처우 | 의료AI 연봉, 헬스케어 데이터 채용, 의료 데이터 사이언티스트 |
| 규제/인허가 | 의료 AI 규제, 의료 AI 인허가 |

### 2.4 블로그 크롤링과의 차이점

| 항목 | 블로그 크롤링 | 뉴스 크롤링 |
|------|---------------|-------------|
| 검색 API | `/v1/search/blog.json` | `/v1/search/news.json` |
| 키워드 수 | 22개 (취업 준비 중심) | 15개 (산업 트렌드 중심) |
| 정렬 기준 | `sim` (정확도순) | `date` (최신순) |
| 링크 필드 | `게시글링크` | `원본링크` + `네이버링크` |
| 날짜 형식 | `YYYYMMDD` (postdate) | RFC 2822 (pubDate) |

## 3. 데이터 스키마

### 3.1 원본 데이터 (data/raw/naver_news_의료AI.csv)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 검색키워드 | str | 수집에 사용된 검색어 | "의료 AI 채용" |
| 제목 | str | 뉴스 기사 제목 | "글로벌 AI, 공격 방어율 57%..." |
| 내용요약 | str | 기사 본문 요약 (description) | "야타브 관계자는..." |
| 원본링크 | str | 원본 언론사 기사 URL | "https://www.sedaily.com/..." |
| 네이버링크 | str | 네이버 뉴스 URL | "https://n.news.naver.com/..." |
| 게시일자 | str | 기사 발행 날짜 (YYYY-MM-DD) | "2026-02-20" |
| 수집일시 | str | 크롤링 시각 | "2026-02-20 19:40:53" |
| 출처 | str | 데이터 소스 | "Naver News" |

### 3.2 전처리 후 데이터 (data/processed/naver_news_의료AI_processed.csv)

| 전처리 항목 | 내용 |
|-------------|------|
| HTML 태그 제거 | `<b>`, `&amp;` 등 HTML 요소 제거 |
| 의료 용어 정규화 | "엑스레이" → "X-ray", "씨티" → "CT" 등 |
| 날짜 형식 통일 | RFC 2822 → `YYYY-MM-DD` |
| 중복 제거 | `원본링크` 기준 URL 중복 제거 |

## 5. 데이터 특성

- **기사 출처**: 서울경제, 매일경제, 한국경제, 전자신문 등 주요 경제/IT 언론사
- **주요 콘텐츠 유형**: 의료 AI 기업 투자/채용 소식, 산업 전망 기사, 정부 정책 뉴스
- **데이터 한계**:
  - API 결과는 기사 제목과 요약(description)만 제공하며, 전체 본문은 포함되지 않음
  - `원본링크`와 `네이버링크` 두 가지 URL이 제공되며, `원본링크`를 중복 제거 기준으로 사용
  - Naver 검색 API 특성상 최대 1,000건까지만 페이징 가능
