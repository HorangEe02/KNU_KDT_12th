# 네이버 블로그 크롤링 (532건)

## 1. 개요

| 항목 | 내용 |
|------|------|
| **데이터 소스** | Naver OpenAPI (블로그 검색) |
| **수집 건수** | 532건 |
| **수집 목적** | 의료 AI 분야 취업 준비 동향 및 커뮤니티 인식 파악 |
| **수집 일시** | 2026-02-20 |
| **API 엔드포인트** | `https://openapi.naver.com/v1/search/blog.json` |

## 2. 수집 방식

### 2.1 사용 기술
- **API**: Naver OpenAPI (검색 > 블로그)
- **인증**: `X-Naver-Client-Id` / `X-Naver-Client-Secret` 헤더 인증
- **구현 파일**: `crawlers/naver_api_crawler.py` > `NaverAPICrawler` 클래스

### 2.2 수집 흐름

```
키워드 리스트 (22개)
    ↓
키워드별 API 호출 (최대 30건/키워드)
    ↓
URL 기반 중복 제거
    ↓
CSV / JSON 저장 (data/raw/)
```

1. 설정 파일(`config/settings.py`)에 정의된 22개 블로그 검색 키워드를 순회
2. 키워드당 최대 30건씩 수집 (API `display=100`, `start` 페이징)
3. `게시글링크` 기준으로 중복 URL 실시간 제거
4. API 호출 간 0.1초 딜레이 적용 (`API_CALL_DELAY`)
5. 결과를 CSV + JSON 형식으로 `data/raw/`에 저장

| 직무 특화 | 의료영상 AI 개발자, 의료 데이터 분석가, DICOM 개발자 |

| 채용/인턴 | 의료 AI 신입 채용, 의료 AI 인턴, 의료 AI 스타트업 취업 |
| 자격/스킬 | 의료 AI 자격증, 헬스케어 데이터 자격증, 의료AI Python 개발 |

## 3. 데이터 스키마

### 3.1 원본 데이터 (data/raw/naver_blog_의료AI.csv)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 검색키워드 | str | 수집에 사용된 검색어 | "의료 AI 취업 준비" |
| 제목 | str | 블로그 게시글 제목 | "의료 AI 부트캠프로 시작하는..." |
| 내용요약 | str | 게시글 본문 요약 (description) | "AI 개발자 취업 과정으로서..." |
| 블로거명 | str | 작성자 블로그 이름 | "꾸르아빠의 생활정보 꿀팁 Note" |
| 블로그링크 | str | 블로거 홈 URL | "blog.naver.com/xxxxx" |
| 게시글링크 | str | 게시글 URL (중복 제거 기준) | "https://blog.naver.com/xxxxx/12345" |
| 게시일자 | str | 게시 날짜 (YYYY-MM-DD) | "2025-08-22" |
| 수집일시 | str | 크롤링 시각 | "2026-02-20 19:40:42" |
| 출처 | str | 데이터 소스 | "Naver Blog" |

### 3.2 전처리 후 데이터 (data/processed/naver_blog_의료AI_processed.csv)

원본 데이터에 다음 전처리가 적용된 결과:

| 전처리 항목 | 내용 |
|-------------|------|
| HTML 태그 제거 | `<b>`, `&amp;` 등 HTML 요소 제거 |
| 의료 용어 정규화 | "엑스레이" → "X-ray", "씨티" → "CT", "엠알아이" → "MRI" 등 |
| 날짜 형식 통일 | 다양한 포맷 → `YYYY-MM-DD` |
| 중복 제거 | `게시글링크` 기준 URL 중복 제거 |

## 4. 데이터 전처리 파이프라인

```
[원본 CSV] ── 660건
    │
    ├─ HTML 태그 제거 (제목, 내용요약)
    ├─ 의료 용어 정규화 (9개 용어 매핑)
    ├─ 날짜 형식 통일 (YYYYMMDD → YYYY-MM-DD)
    └─ URL 기반 중복 제거
    │
[전처리 CSV] ── 532건
```

- **구현 파일**: `processors/data_cleaner.py` > `DataCleaner` 클래스
- 의료 용어 매핑: X-ray, CT, MRI, PACS, 초음파, DICOM, EMR, HL7, FHIR (9종)

## 6. 데이터 특성

- **게시 기간**: 2024년 ~ 2026년 2월 (약 2년간)
- **주요 콘텐츠 유형**: 국비지원 부트캠프 홍보, 취업 준비 후기, 교육과정 소개, AI 학과 정보
- **데이터 한계**:
  - API 결과는 블로그 게시글의 제목과 요약(description)만 제공하며, 전체 본문은 포함되지 않음
  - 키워드당 최대 30건으로 제한하여 수집량 조절
  - Naver 검색 API 특성상 최대 1,000건까지만 페이징 가능
