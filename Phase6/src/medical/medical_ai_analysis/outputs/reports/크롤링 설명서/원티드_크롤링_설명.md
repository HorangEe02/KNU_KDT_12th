## 2. 수집 방식

### 2.1 사용 기술
- **API**: Wanted API v4 (`https://www.wanted.co.kr/api/v4/jobs`)
- **HTTP 클라이언트**: `requests.Session`
- **구현 파일**: `crawlers/wanted_crawler.py` > `WantedCrawler` 클래스
- **Selenium 불필요**: API 기반으로 직접 데이터 수집

### 2.2 수집 흐름 (2단계 방식)

```
[1단계] 목록 API 호출
    카테고리: 의료/제약/바이오 (tag_type_ids=515)
    → offset 페이징으로 전체 포지션 ID 수집
    ↓
[2단계] 상세 API 호출
    각 포지션 ID별 상세 정보 수집
    → 주요업무, 자격요건, 우대사항, 기술스택 등
    ↓
목록 + 상세 데이터 통합
    ↓
CSV 저장 (data/raw/)
```

### 2.3 1단계: 목록 수집

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `country` | `kr` | 한국 |
| `tag_type_ids` | `515` | 의료/제약/바이오 카테고리 |
| `job_sort` | `job.popularity_order` | 인기순 정렬 |
| `limit` | `20` | 페이지당 20건 |
| `offset` | `0, 20, 40, ...` | 페이징 |

- `links.next`가 없을 때까지 반복 호출
- 호출 간 0.3초 딜레이

### 2.4 2단계: 상세 수집

- API 엔드포인트: `https://www.wanted.co.kr/api/v4/jobs/{job_id}`
- 각 포지션의 상세 정보 (주요업무, 자격요건, 우대사항, 혜택, 기술스택 태그 등) 수집
- 호출 간 0.3초 딜레이

## 6. 데이터 특성

- **카테고리 특화**: 키워드 검색이 아닌 의료/제약/바이오 카테고리(515) 전체를 수집하여 누락 최소화
- **상세 정보 풍부**: API를 통해 자격요건, 우대사항, 주요업무, 기술스택 태그 등 상세 텍스트 수집
- **데이터 한계**:
  - 의료/제약/바이오 카테고리에 등록된 포지션만 수집 (다른 카테고리에 등록된 의료 AI 포지션은 미포함)
  - 연봉 정보는 직접 제공되지 않으며, `연봉범위` 필드는 경력 연차 범위를 나타냄
  - 포지션 수가 상대적으로 적음 (75건) - 원티드 의료/제약/바이오 카테고리 규모 자체가 작음
