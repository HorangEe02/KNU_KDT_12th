## 1. 프로젝트 개요

### 1.1 목적

의료 AI 분야 취업을 준비하는 구직자를 위해, 실제 채용 시장 데이터를 수집하고 분석하여 **필수 기술스택**, **기업 평판**, **면접 동향**, **산업 트렌드** 등의 인사이트를 도출한다.

### 1.2 분석 파이프라인

```
크롤링 (Crawling) → 전처리 (Processing) → 분석 (Analysis) → 시각화 (Visualization) → URL 정리
```

| 단계 | 도구 | 설명 |
|------|------|------|
| 크롤링 | Naver OpenAPI, requests+BS4, 원티드 API, Selenium | 블로그/뉴스 API + 사람인 검색카드 크롤링 + 원티드 API + 잡플래닛 웹 크롤링 |
| 전처리 | pandas | 중복 제거, 텍스트 정제, 날짜 통일 |
| 분석 | KoNLPy(Okt), 정규식 | 형태소 분석, 기술스택/자격증/도메인 빈도 |
| 시각화 | matplotlib, seaborn, Plotly, WordCloud | 정적 차트 + 인터랙티브 + 워드클라우드 |

## 5. 종합 결론 및 제언

### 5.1 의료 AI 취업 시장 특징

1. **Python이 핵심 기술**: 사람인 공고제목/직무분야까지 반영한 결과, Python(790건)이 압도적 1위. 의료 AI 개발의 사실상 표준 언어
2. **딥러닝/머신러닝 필수**: 딥러닝(598건)·머신러닝(585건)이 2~3위. AI/ML 기술은 기본 요건
3. **도메인 전문성이 차별화 요인**: 의료영상(245건), 임상시험(195건), FDA(51건) 등 도메인 지식이 여전히 높은 빈도 — 코딩만으로는 불충분
4. **통계 역량 동시 요구**: R(535건), SPSS(48건), SAS(26건) 등 통계 도구가 상위권 — FDA 인허가를 위한 임상 통계 분석 필수
5. **인허가 지식 필수**: 인허가(124건), SaMD(70건), FDA(51건) 등 인허가 관련 키워드가 도메인의 24% 차지 — 규제 대응 능력이 차별화 요인
6. **의료 면허 + IT 융합 수요**: 방사선사(34건)가 자격증 1위(46.58%) — 의료 현장 경험 + AI 역량 조합 인재에 대한 높은 수요
7. **스타트업 중심 시장**: 12개 대상 기업 중 7개(58%)가 스타트업 - 빠른 성장과 도전적 환경

### 5.2 취업 준비자를 위한 제언

**필수 기술 역량**:
- Python 기반 데이터 분석 및 머신러닝/딥러닝 (PyTorch, TensorFlow)
- R/SPSS/SAS 기반 통계 분석 — 임상시험 데이터 분석의 업계 표준 도구
- SQL, AWS 등 실무 개발 인프라 기술
- 의료영상 처리 (DICOM, OpenCV 등)

**차별화 역량**:
- FDA/MFDS 인허가 프로세스 이해 (SaMD, 510(k))
- 임상시험 통계 설계 및 분석 경험
- 논문 작성/특허 출원 경험
- 의료 도메인 지식 (영상의학, 병리, 임상시험)

**추천 자격증**:
- 방사선사/임상병리사 - 의료 도메인 전문성 (해당 전공자)
- ADP (데이터분석전문가) - 데이터 분석 역량 증명
- 정보처리기사, ADsP, SQLD - 기본 IT 역량 증명
- 의공기사 - 의공학 배경 인재

**기업 선택 시 고려사항**:
- 대기업(네이버클라우드, 카카오헬스케어)은 높은 만족도(4.87, 3.67) - 안정적 환경 선호 시
- 스타트업은 빠른 성장 기회가 있으나 만족도는 상대적으로 낮음(평균 2.7)
- 면접 난이도는 대부분 '보통' 수준(67%) - 기본기에 충실한 준비 필요

### 5.3 한계점 및 향후 과제

| 한계점 | 설명 |
|--------|------|
| 잡플래닛 멤버십 제한 | 리뷰 본문(장점/단점)과 면접 상세 경험은 멤버십 벽으로 수집 불가 - 메타데이터만 수집 |
| 사람인 상세 페이지 제한 | 상세 페이지가 JS 렌더링으로 접근 제한 - 검색 결과 카드 기반 데이터만 수집 |
| 시간 범위 제한 | 수집 시점 기준 스냅샷 - 시계열 변화 추적 불가 |
| 표본 편향 | 잡플래닛 리뷰 작성자 특성상 부정적 리뷰 편향 가능성 |

**향후 과제**:
- 정기적 크롤링 자동화 → 시계열 트렌드 분석
- 감성 분석(Sentiment Analysis) 적용 → 리뷰 텍스트의 긍/부정 정량화
- LLM 기반 면접 질문 패턴 분석 → 면접 준비 가이드 생성
- 사람인 상세 페이지 Selenium 기반 크롤링 추가 → 상세 직무기술서 분석
