# 기(起) — 프로젝트 배경과 동기

## 1. 왜 이 프로젝트를 수행하게 되었는가

### 1.1 시대적 배경

기술의 고도화와 산업 구조의 급변 속에서 취업 시장은 점점 더 복잡해지고 있다. 단순히 "좋은 학력"이나 "전공 일치"만으로는 취업 경쟁력을 확보하기 어려운 시대가 되었다. 특히 AI, 모빌리티, 스마트시티 등 첨단 산업 분야에서는 **"어떤 역량을 갖춰야 하는가"**라는 질문에 대한 답이 매년, 매분기 달라지고 있다.

이러한 환경에서 **데이터 기반의 취업 역량 분석**이 필요하다는 공통된 문제의식 아래, K-Digital Training 과정의 팀 프로젝트로 본 미니프로젝트가 기획되었다.

### 1.2 팀 구성과 역할 분담

Team 5는 "모빌리티·스마트시티 산업의 인재 역량"이라는 대주제 아래, 4개 세부 분야를 각 팀원이 독립적으로 분석하는 구조로 진행되었다.

| 순번 | 팀원             | 담당 분야         | 분석 대상                                  |
| :--: | ---------------- | ----------------- | ------------------------------------------ |
|  1  | 박건희           | 자동차 생산       | 자동차 부품·생산 기업 채용 동향           |
|  2  | 정유진           | 자동차 모빌리티   | 자율주행·모빌리티 서비스 채용 동향        |
|  3  | 박성훈           | 비즈니스          | 비즈니스·마케팅 직군 채용 동향            |
|  4  | **박준영** | **의료 AI** | **의료 AI 분야 취업 시장 정량 분석** |

### 1.3 의료 AI 분야를 선택한 이유

의료 AI는 영상의학, 디지털 헬스케어, 신약 개발, 원격 진료 등 다양한 의료 분야에서 급속히 성장하고 있는 분야이다. 그러나 이 분야의 채용 시장에 대한 **체계적인 데이터 분석**은 부족한 실정이었다. "의료 AI 개발자가 되려면 어떤 기술을 익혀야 하는가?", "어떤 자격증이 실제로 도움이 되는가?", "기업들의 근무 환경은 어떠한가?" 같은 질문에 감이 아닌 **데이터로 답할 수 있는 분석 체계**를 구축하고자 했다.

### 1.4 프로젝트의 목표

1. **정량적 데이터 수집**: 4개 플랫폼에서 의료 AI 관련 데이터를 자동 수집
2. **다각도 분석**: 기술스택, 자격증, 의료 도메인, 채용 구조, 기업 환경을 종합 분석
3. **시각화 기반 인사이트 도출**: 70종 이상의 차트로 분석 결과를 직관적으로 전달
4. **재사용 가능한 파이프라인 구축**: 다른 분야에도 적용 가능한 모듈화된 분석 프레임워크 설계

# 승(承) — 프로젝트 수행 방법론

## 2. 어떤 방식으로 프로젝트를 수행했는가

### 2.1 전체 아키텍처: 5단계 파이프라인

프로젝트는 **엔드투엔드 자동화 파이프라인**으로 설계하였다. 데이터 수집부터 시각화까지 5개 Phase를 순차적으로 실행하는 구조이다.

```
[Phase 1] 크롤링     →  [Phase 2] 전처리     →  [Phase 3] 분석
 4개 소스               HTML 제거                형태소 분석
 3,532건 수집           의료용어 정규화           기술스택/자격증 빈도
                        중복 제거                 도메인 분류

     →  [Phase 4] 시각화     →  [Phase 5] URL 정리
         10개 모듈               3,262개 고유 URL
         70종 차트               CSV 출력
```

### 2.2 데이터 수집 전략 (Phase 1)

4개 서로 다른 플랫폼에서, 각각의 특성에 맞는 크롤링 기법을 적용하였다.

| 데이터 소스             | 수집 방법                               |     수집 건수     | 수집 항목                                |
| ----------------------- | --------------------------------------- | :---------------: | ---------------------------------------- |
| **네이버 블로그** | Naver OpenAPI (22개 키워드)             |       660건       | 의료 AI 취업 관련 블로그 포스팅          |
| **네이버 뉴스**   | Naver OpenAPI (15개 키워드)             |       451건       | 의료 AI 채용·산업 관련 뉴스 기사        |
| **사람인**        | requests + BeautifulSoup (12개 키워드)  |      2,078건      | 의료 AI 분야 채용공고                    |
| **원티드**        | REST API v4 (카테고리 515)              |       75건       | 의료/제약/바이오 카테고리 포지션 상세    |
| **잡플래닛**      | Selenium WebDriver + 로그인 (12개 기업) |       268건       | 기업정보(12) + 리뷰(155) + 면접후기(101) |
| **합계**          |                                         | **3,532건** |                                          |

#### 크롤링 기술 선택 이유

- **Naver OpenAPI**: 네이버 블로그/뉴스는 공식 API를 제공하므로, 안정적이고 구조화된 데이터 수집이 가능하다. API 키 기반 인증으로 일 25,000건까지 호출 가능.
- **requests + BeautifulSoup**: 사람인은 서버 사이드 렌더링(SSR) 방식이므로, 별도의 브라우저 자동화 없이 HTTP 요청과 DOM 파싱만으로 충분하다. User-Agent 헤더와 랜덤 딜레이(2~5초)로 안티봇 대응.
- **REST API v4**: 원티드는 공개 REST API를 제공하며, 목록 API → 상세 API의 2단계 수집으로 포지션별 상세 정보를 확보.
- **Selenium WebDriver**: 잡플래닛은 로그인 필수 + 동적 렌더링 페이지로, 브라우저 자동화가 불가피하다. WebDriver 탐지 우회 처리 적용.

### 2.3 데이터 전처리 (Phase 2)

수집된 원시 데이터 3,532건을 정제하여 3,404건의 분석용 데이터셋을 구축하였다.

| 전처리 항목                | 적용 내용                                                     | 처리 건수                     |
| -------------------------- | ------------------------------------------------------------- | ----------------------------- |
| **중복 제거**        | URL 기반 중복 판별                                            | 블로그 128건, 원티드 1건 제거 |
| **HTML 태그 제거**   | 정규표현식 기반 HTML/특수문자 정리                            | 전체 텍스트 필드              |
| **의료 용어 정규화** | X-ray, CT, MRI, PACS, DICOM, EMR, HL7, FHIR, 초음파 표기 통일 | 9개 핵심 용어                 |
| **날짜 형식 통일**   | RFC 2822, 한국어 표기 → YYYY-MM-DD 변환                      | 전체 날짜 필드                |

### 2.4 텍스트 분석 (Phase 3)

- **형태소 분석기**: KoNLPy의 Okt(Open Korean Text)를 사용하여 명사 추출
- **불용어 처리**: 조사, 접속사, 범용 동사 등 38개 불용어 사전 적용
- **기술스택 추출**: 50개 이상의 기술 키워드 사전 기반 정규표현식 매칭
- **자격증 추출**: 12개 자격증 키워드 기반 빈도 분석
- **의료 도메인 분류**: 5개 카테고리(영상의학, 인허가, 임상, 병리, 유전체), 21개 세부 키워드

### 3.7 핵심 발견 6: 콘텐츠 트렌드

## 5. 느낀 점

### 5.1 데이터 크롤링의 현실

이론적으로는 단순해 보이는 웹 크롤링이 실제로는 **각 플랫폼마다 완전히 다른 접근법**을 요구한다는 점을 체감하였다. 네이버는 API, 사람인은 HTTP 파싱, 원티드는 REST API, 잡플래닛은 브라우저 자동화가 각각 필요했으며, 안티봇 대응(User-Agent, 랜덤 딜레이, WebDriver 탐지 우회)까지 고려해야 했다. **하나의 만능 크롤러는 존재하지 않는다**는 것을 직접 경험하였다.

### 5.2 데이터 분석의 힘

3,404건의 데이터를 분석하기 전에는 "의료 AI면 딥러닝이 중요하겠지" 정도의 막연한 인식만 있었다. 그러나 실제 데이터를 들여다보니, **R이 11.5%로 4위를 차지**하고, **방사선사 면허가 IT 자격증보다 압도적으로 높은 수요**를 보이며, **SaMD 인허가가 도메인의 24%**를 차지하는 등 예상치 못한 인사이트가 많았다. 감이 아닌 데이터로 의사결정하는 것의 가치를 실감하였다.

### 5.3 시각화의 중요성

70종의 차트를 생성하면서, **같은 데이터라도 어떤 차트로 표현하느냐에 따라 전달되는 메시지가 완전히 달라진다**는 점을 경험하였다. 단순 막대 차트로는 보이지 않던 패턴이 히트맵이나 네트워크 그래프에서 드러나고, 인터랙티브 차트(Plotly HTML)를 통해 발표 시 청중의 참여를 유도할 수 있었다.

### 5.4 AI 보조 도구(Claude Code)의 활용

프로젝트 전반에 걸쳐 **Claude Code**를 활용하여 코드 생성 및 디버깅을 수행하였다. 특히 크롤러 구현, 시각화 코드 작성, 보고서 생성 등에서 생산성을 크게 높일 수 있었다. AI 도구를 단순히 사용하는 것에 그치지 않고, **프로젝트 구조 설계와 가이드라인 문서 작성을 통해 AI가 효과적으로 작동하는 환경을 구축하는 것**이 핵심이라는 점을 배웠다.

### 5.5 모듈화와 재사용성의 가치

프로젝트를 진행하면서, 단순히 의료 AI 분석에 그치지 않고 **다른 분야에도 적용 가능한 프레임워크**로 설계한 것이 큰 의미가 있었다. 실제로 `settings.py` 하나만 변경하면 80%의 전환이 완료되는 구조를 만들었고, 이를 바탕으로 팀원들에게도 적용할 수 있는 **8개의 가이드라인 문서(00~07번)**를 작성하였다. 이 경험을 통해 "일회성 분석"이 아닌 "재사용 가능한 시스템"을 만드는 것의 중요성을 깨달았다.
