# 📋 Streamlit 기반 Enhanced WMS 대시보드 세부 기능 설명서

## 1. 개요

* **프로젝트명:** 머신러닝 기반 식료품 유통 재고 관리 최적화 시스템
* **대시보드 목적:** WMS 도입률이 낮은 중소·중견 식료품 유통업체를 위한 데이터 기반 재고 관리 의사결정 지원 시스템 시연 (PoC)
* **주요 기술:** Streamlit, Python (모델 연동, 전처리, 시각화), Plotly

## 2. 공통 구성 요소 및 글로벌 필터

이 영역은 모든 페이지에서 고정되어 나타나며, 전체 데이터에 대한 필터링과 원본 대 보강 데이터의 비교를 제어합니다.

## 3. 핵심 기능 페이지 세부 명세

### 📊 종합 대시보드 (Dashboard Overview)

* **목적:** 원본 vs 보강 데이터 비교를 통한 통합적 분석 결과 및 글로벌 벤치마크 패턴 대조 제시.

| 기능명                         | 데이터 소스                                  | UI 요소             | 상세 설명                                                                                                    |
| :----------------------------- | :------------------------------------------- | :------------------ | :----------------------------------------------------------------------------------------------------------- |
| **핵심 지표 (Metrics)**  | 파생 변수 (`Enhanced_E-Grocery.csv`)       | `st.metric`       | 총 SKU 수, 품절 위험 품목 수, 폐기 위험 품목 수, 권장 발주 금액 표출. 글로벌 토글 상태에 따라 수치 업데이트. |
| **글로벌 벤치마크 비교** | `원본 E-Grocery` + `Global Food Wastage` | `st.plotly_chart` | 원본 카테고리별 폐기율과 보강 데이터의 평균 폐기율을 겹쳐 그린 바 차트로 글로벌 패턴과의 대조 제시.          |
| **종합 상황 시각화**     | 파생 변수 (`Inventory_Status`)             | `st.plotly_chart` | 카테고리별 재고 상태 (In Stock, Low Stock, Excess Stock 등) 비율 시각화.                                     |

### 📦 상태 진단 (Subtopic 1 — 분류)

* **목적:** 공급망 리스크(백오더, 물류 지연)가 반영된 정교한 재고 상태 확인.

| 기능명                     | 데이터 소스                                                      | UI 요소             | 상세 설명                                                                       |
| :------------------------- | :--------------------------------------------------------------- | :------------------ | :------------------------------------------------------------------------------ |
| **상태 필터링**      | `원본 E-Grocery`                                               | `st.selectbox`    | Low Stock, Expiring Soon, In Stock, Out of Stock 중 조회할 상태 선택.           |
| **위험 감지 테이블** | `Enhanced_E-Grocery.csv` (`Inventory_Status == 'Low Stock'`) | `st.dataframe`    | 새롭게 설계된 `Backorder_Risk_Score`(백오더 위험도)가 높은 순서대로 리스트업. |

### 📈 수요 예측 시뮬레이터 (Subtopic 2 — 회귀)

* **목적:** 매장 특성, 가격 전략 등이 반영된 정교한 판매량 시뮬레이션 및 판단 근거 제시.

| 기능명                     | 데이터 소스                                  | UI 요소                         | 상세 설명                                                                                                               |

## 4. 기술적 특징

* **데이터 파이프라인 분리:** 주피터 노트북에서 전처리 및 보강 데이터 매핑 완료 후 최종 병합본 CSV 파일(`Enhanced_E-Grocery.csv`) 하나만 업로드하여 대시보드 구동. 실시간 전처리에 따른 버벅거림 방지.
* **캐싱 적용:** `@st.cache_data` (데이터 로딩), `@st.cache_resource` (모델 로딩) 데코레이터를 사용하여 웹 페이지 로딩 속도 최적화.
* **모델 관리:** 학습된 pkl/joblib 모델 로드하여 실시간 시뮬레이션 구동.
