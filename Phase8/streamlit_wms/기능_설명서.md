## 2. 듀얼 모드 시스템

### 개요
모든 분석 페이지(대시보드, 데이터 탐색기, 재고 상태 분류, 판매량 예측, 폐기 위험 탐지, 발주 전략)는 **WMS 시뮬레이터** / **알고리즘 인사이트** 두 가지 모드를 제공한다. 각 페이지 상단에 토글 버튼이 배치되어 있으며, 페이지별로 독립적인 모드 상태를 유지한다.

### 모드 비교

| 항목 | WMS 시뮬레이터 | 알고리즘 인사이트 |
|------|---------------|------------------|
| 대상 사용자 | 현장 운영자, 관리자 | 데이터 분석가, 개발자 |
| 초점 | 핵심 지표 요약, 시뮬레이션 | ML 모델 상세 분석, 알고리즘 비교 |
| 정보량 | 간결한 KPI + 액션 중심 | 상세 차트, 통계, 모델 성능 지표 |
| 기본값 | - | 알고리즘 인사이트 (기본 선택) |

### 동작 방식
- 각 페이지는 `st.session_state`에 페이지별 모드 키를 저장 (예: `dash_mode`, `explorer_mode`, `inventory_mode`, `sales_mode`, `waste_mode`, `reorder_mode`)
- 모드 전환 시 해당 페이지의 콘텐츠가 즉시 전환되며, 다른 페이지의 모드에는 영향을 주지 않음
- **WMS 시뮬레이터 모드:** 간소화된 KPI 카드, 긴급 현황 위주의 테이블, 제품별 시뮬레이션 인터페이스
- **알고리즘 인사이트 모드:** 전체 차트, 혼동 행렬, 오차 분포, 피처 중요도, 알고리즘 비교 등 ML 분석 상세 콘텐츠

### 페이지별 모드 라벨

| 페이지 | WMS 시뮬레이터 라벨 | 알고리즘 인사이트 라벨 |
|--------|---------------------|----------------------|
| 대시보드 | 핵심 지표 요약 | ML 기반 KPI 요약 및 재고 현황 분석 |
| 폐기 위험 탐지 | 폐기 위험 제품 현황 | ML 폐기 위험 탐지, 정확도 98.8% |
| 발주 전략 | 긴급 발주 현황, 제품별 시뮬레이션, 발주 해석 가이드 | 제품 그룹 분석, Sub-clustering, 알고리즘 비교, Safety Stock EOQ, 민감도 분석, 비선형 시각화 |

## 3. 대시보드

**파일:** `pages/1_Dashboard.py`

### 개요
전체 재고 현황을 KPI, 차트, 시뮬레이터로 요약하는 종합 대시보드이다.

### KPI 카드 (상단 4개)

| KPI | 의미 | 계산 방식 |
|-----|------|-----------|
| 전체 제품 수 | 관리 중인 SKU 총 수 | 데이터 행 수 |
| 재고 부족 위험 | 재고 부족 또는 품절 제품 수 | Inventory_Status가 Low Stock 또는 Out of Stock인 항목 |
| 폐기 위험 높음 | 유통기한 초과 위험 제품 수 | ML 모델이 위험(1)으로 예측한 항목 |
| 최적 주문 총액 | 전체 EOQ 금액 합계 | 모든 SKU의 최적 주문량 합산 |

### 차트 및 시각화

#### (1) 제품 그룹 3D 분석 (왼쪽)
- **차트 유형:** 3D 산점도
- **축 구성:**
  - X축: 현재 보유 수량 (Quantity_On_Hand)
  - Y축: 일평균 판매량 (Avg_Daily_Sales)
  - Z축: 재고 소진 예상일 (Days_of_Inventory)
- **색상 구분:** K-Means 클러스터 결과
  - 빠른 회전 제품 — 판매 속도가 빠른 제품군
  - 느린 회전 제품 — 판매 속도가 느린 제품군
  - 계절성 제품 — 계절에 따라 판매량이 변하는 제품군

#### (2) 판매량 예측 시뮬레이터 (오른쪽)
- **기능:** 개별 제품의 조건을 바꿔가며 예측 판매량 변화를 확인
- **조작 가능 항목:**
  - 제품 선택 (드롭다운)
  - 현재 보유 수량 (슬라이더, 10~1,000)
  - 배송 소요일 (슬라이더, 1~14일)
  - 자동 발주 기준선 (슬라이더, 10~800)
- **출력:** XGBoost 모델의 예측 일일 판매량

#### (3) 위험 제품 현황 (하단 왼쪽)
- 폐기 위험이 높은 상위 30개 제품 목록
- 위험도(Risk_Prob)에 따라 빨간색 그라데이션 표시
- 표시 정보: 제품 ID, 이름, 카테고리, 유통기한 잔여일, 위험 확률, 보유 수량

#### (4) 재고 상태 분포 (하단 중앙)
- **차트 유형:** 도넛 차트
- 재고 상태별 비율: 충분(Sufficient) / 부족(Low Stock) / 품절(Out of Stock) 등

#### (5) 중요도 등급별 재고 가치 (하단 오른쪽)
- **차트 유형:** 막대 차트
- ABC 등급(A/B/C)별 재고 금액 합계

## 4. 데이터 탐색기

**파일:** `pages/2_Data_Explorer.py`

### 개요
원본 데이터를 자유롭게 필터링, 검색, 통계 분석할 수 있는 탐색 도구이다.

### 사이드바 필터

| 필터 | 설명 | 기본값 |
|------|------|--------|
| 카테고리 | 식품 분류 (유제품, 냉동식품 등) | 전체 선택 |
| 중요도 등급 (ABC) | A=매출 상위 80%, B=중간 15%, C=하위 5% | 전체 선택 |
| 재고 상태 | 충분 / 부족 / 품절 등 | 전체 선택 |
| 폐기 위험 | 전체 / Risk / Safe | 전체 |
| 제품 검색 | SKU ID 또는 제품명으로 검색 | 빈칸 |

### KPI 카드 (상단 4개)

| KPI | 의미 |
|-----|------|
| 필터 결과 | 현재 조건에 맞는 제품 수 |
| 전체 제품 수 | 전체 SKU 수 |
| 평균 일일 판매량 | 필터된 제품의 하루 평균 판매 수량 |
| 재고 총 가치 | 필터된 제품의 재고 금액 합계 |

### 주요 기능

#### (1) 데이터 테이블
- 15개 핵심 컬럼을 표시하는 정렬 가능한 테이블
- SKU ID 기준 정렬

#### (2) 기술 통계
- 9개 수치형 변수의 기술 통계 (평균, 표준편차, 최소, 최대, 사분위수)

#### (3) 분포 시각화
- 분석 항목 선택 후 두 가지 차트 동시 표시:
  - **히스토그램:** ABC 등급별 색상 구분, 30개 구간
  - **박스플롯:** 카테고리별 분포 비교

#### (4) 상관관계 히트맵
- 선택한 변수들 간의 상관계수를 색상으로 표시
- 빨간색 = 양의 상관 / 파란색 = 음의 상관
- 기본 4개 변수: 보유 수량, 일평균 판매량, 재고 소진일, 단가

## 5. 재고 상태 분류

**파일:** `pages/3_Inventory_Status.py`

### 개요
제품의 재고 상태(충분/부족/품절 등)를 LightGBM 모델로 자동 분류한다. 정확도 98.8%.

### KPI 카드 (상단 4개)

| KPI | 의미 |
|-----|------|
| 전체 정확도 | 모델이 올바르게 예측한 비율 |
| 전체 제품 수 | 분석 대상 SKU 수 |
| 상태 유형 수 | 구분되는 재고 상태 종류 (충분/부족/품절 등) |
| 잘못 분류된 제품 | 실제와 다르게 예측한 제품 수 |

### 탭 구성

#### 탭 1: 분류 현황
- **실제 vs 예측 분포:** 실제 상태와 예측 상태의 건수 비교 (표 + 그룹 막대 차트)
- **카테고리별 재고 상태:** 식품 카테고리별 상태 분포 (누적 막대)
- **ABC 등급별 재고 상태:** 중요도 등급별 상태 분포 (누적 막대)

#### 탭 2: 예측 정확도 분석
- **혼동 행렬 (히트맵):**
  - X축: 예측값, Y축: 실제값
  - 대각선 숫자가 클수록 정확한 예측
  - Blues 색상 스케일
- **오분류 SKU 목록:** 실제와 예측이 다른 제품들의 상세 정보

#### 탭 3: 개별 제품 예측
- 제품 선택 후 상세 정보 표시:
  - 제품명, 카테고리 (ABC 등급 포함), 창고 위치
  - 현재 보유 수량, 자동 발주 기준선, 재고 소진 예상일
- **판정 결과:** 실제 상태 vs 예측 상태 일치/불일치 표시
- **클래스별 확률 막대 차트:** 각 상태로 분류될 확률 시각화

## 6. 판매량 예측 시뮬레이터

**파일:** `pages/4_Sales_Prediction.py`

### 개요
XGBoost 회귀 모델로 제품별 하루 평균 판매량을 예측한다. R²=0.9477, RMSE=4.79.

### KPI 카드 (상단 4개)

| KPI | 의미 | 해석 |
|-----|------|------|
| 예측 정확도 (R²) | 모델이 실제 판매량을 설명하는 비율 | 1에 가까울수록 좋음 |
| 평균 예측 오차 (RMSE) | 예측값과 실제값의 평균 차이 | 낮을수록 정확 |
| 평균 오차 크기 (MAE) | 절대값 기준 평균 오차 | 낮을수록 좋음 |
| 평균 일일 판매량 | 전체 제품의 하루 평균 판매 수량 | 기준값 |

### 탭 구성

#### 탭 1: 실제 vs 예측 비교
- **산점도:** 실제 판매량(X) vs 예측 판매량(Y)
  - 점선 = 이상적 예측선 (완벽하면 모든 점이 이 선 위)
  - ABC 등급별 색상 구분 (A=빨강, B=주황, C=파랑)
- **오차 분포 히스토그램:** 예측 오차가 0 근처에 몰려 있을수록 좋음
- **오차 비대칭 분석:**
  - 과대 예측 (실제보다 높게 예측) 건수
  - 과소 예측 (실제보다 낮게 예측) 건수
  - 정확 예측 (|오차| < 2) 건수

#### 탭 2: 판매량 구간별 성능
- **구간 분류:**
  - 저판매 (0~20) / 중판매 (20~40) / 고판매 (40~70) / 초고판매 (70+)
- **구간별 성능 테이블:** 각 구간의 RMSE, MAE, Bias
- **카테고리별 성능 테이블:** 식품 카테고리별 예측 정확도

#### 탭 3: 개별 제품 예측
- 제품 선택 후 상세 정보:
  - 실제 일일 판매량 vs 예측 일일 판매량 vs 예측 오차
  - 재고 소진 예상일
- **조건 변경 시뮬레이션:**
  - 보유 수량 / 발주 기준선 / 배송 소요일 슬라이더 조작
  - 조건 변경 시 예측 판매량이 어떻게 달라지는지 실시간 확인
  - 기존 대비 변화량 표시

## 7. 폐기 위험 탐지

**파일:** `pages/5_Waste_Risk.py`

### 개요
유통기한 초과로 폐기될 위험이 있는 제품을 ML 모델(Logistic Regression)로 탐지한다. 정확도 98.8%.

### KPI 카드 (상단 4개)

| KPI | 의미 |
|-----|------|
| 위험 제품 수 | ML 모델이 폐기 위험으로 예측한 제품 수 |
| 안전 제품 수 | 폐기 위험이 낮은 제품 수 |
| 위험 비율 | 전체 중 위험 제품 비율 (%) |
| 위험 재고 금액 | 폐기 시 예상 손실 금액 ($) |

### 탭 구성

#### 탭 1: 위험 제품 목록
- **필터:** 카테고리 선택, 정렬 기준 (위험도 / 잔여일 / 금액)
- **테이블:** 위험 제품 목록 (위험도에 빨간색 그라데이션)
- **차트:**
  - 카테고리별 위험 제품 수 (막대)
  - 유통기한 잔여일 vs 위험도 산점도 (원 크기 = 재고 금액)

#### 탭 2: 신선식품 분석
- **신선식품 카테고리 상세:** 카테고리별 위험/안전 제품 수와 위험 비율
- **출하 방식별 위험 분석:**
  - FIFO (선입선출) vs FEFO (유통기한순 출고)별 위험률 비교

#### 탭 3: 개별 제품 예측
- 제품 선택 후 상세 정보 표시:
  - 제품명, 카테고리, 출하 방식
  - 보유 수량, 하루 평균 판매량, 유통기한 잔여일, 재고 금액
- **위험도 게이지 차트:** 0~100% 스케일, 3단계 색상 (녹/주황/빨강)
- **주요 위험 요인 분석:**
  - 유통기한 30일 미만
  - 부패성 카테고리 여부
  - FIFO 정책 사용 시 FEFO 전환 권장
  - 손상 수량 존재
  - 낮은 판매 속도 (5개/일 미만)

## 8. 발주 전략

**파일:** `pages/6_Reorder_Strategy.py`

### 개요
재고 소진일 예측(XGBoost) + 제품 그룹 분류(K-Means + Sub-clustering) + DBSCAN/GMM 비교 + Safety Stock 통합 EOQ + S×H 민감도 분석 + t-SNE/UMAP 비선형 시각화를 결합한 종합 발주 전략 페이지이다.

**v3.5 주요 변경:** v2.0의 3탭 → 9탭 확장 (알고리즘 인사이트 모드). Sub-clustering 기반 3-Tier 분류, 다중 알고리즘 비교, 서비스 수준별 Safety Stock 시뮬레이션, Phase A 회귀 개선, AI 모델 분석, 용어 사전 탭 추가.

### KPI 카드 (상단 4개)

| KPI | 의미 |
|-----|------|
| 평균 재고 소진일 | 현재 속도로 재고가 며칠 후 바닥나는지 |
| 평균 발주 여유도 | 양수 = 여유 / 음수 = 긴급 발주 필요 |
| 긴급 발주 제품 | 발주 여유도가 음수인 제품 수 |
| 관리 등급 수 | 3-Tier 분류 체계 (일반/준긴급/긴급) |

### 탭 구성 (알고리즘 인사이트 모드: 9탭)

#### 탭 1: 발주 긴급도
- **계산 공식:**
  ```
  발주 여유도 = (보유 수량 - 발주 기준선) / 안전 재고
  ```
- **긴급도 등급:**
  | 등급 | 범위 | 의미 |
  |------|------|------|
  | 매우 긴급 | < -1 | 재고 심각 부족 |
  | 긴급 | -1 ~ 0 | 발주 기준선 이하 |
  | 적정 | 0 ~ 1 | 적정 수준 |
  | 여유 | > 1 | 충분한 재고 |
- **원형 차트:** 등급별 제품 비율
- **히스토그램:** 발주 여유도 분포 (빨간 점선 = 발주 기준선)
- **긴급 발주 목록:** 여유도 음수 제품의 상세 테이블 (빨간 그라데이션)

#### 탭 2: 제품 그룹 · 3-Tier
- **3-Tier 분류 체계:** Cluster 1 내부를 Sub-clustering(K=2)하여 3단계 분류
  | Tier | 구성 | 의미 |
  |------|------|------|
  | Tier1: 일반 관리 | Cluster 0 전체 (~80%) | 표준 발주 프로세스 |
  | Tier2: 준긴급 | Cluster 1의 Sub-cluster 0 (~18%) | 주의 관찰 필요 |
  | Tier3: 긴급 | Cluster 1의 Sub-cluster 1 (~2%) | 즉시 보충 필요 |
- **도넛 차트:** Tier별 제품 비율
- **산점도:** 보유 수량 vs 재고 소진일 (Tier별 색상)
- **Tier별 통계:** SKU 수, 평균 QOH, DOI, EOQ, Safety Stock, Lead Time, 긴급 비율
- **Sub-clustering 메타데이터:** Hopkins Statistic, 최적 K, Silhouette Score
- **ABC·카테고리 분포:** Tier별 ABC 등급 및 식품 카테고리 구성 막대 차트

#### 탭 3: Safety Stock EOQ
- **Safety Stock 공식:** `SS = Z × σ_d × √(LT)`
  - Z: 서비스 수준에 대응하는 정규분포 Z값
  - σ_d: 카테고리 내 일평균 판매량 표준편차
  - LT: 배송 소요일 (Lead Time)
- **통합 TC 공식:** `TC = (D/Q)×S + (Q/2 + SS)×H`
- **서비스 수준별 시뮬레이션:**
  | 서비스 수준 | Z값 | 의미 |
  |-------------|------|------|
  | 90% | 1.282 | 10% 결품 허용 |
  | 95% | 1.645 | 5% 결품 허용 |
  | 99% | 2.326 | 1% 결품 허용 |
- **조작 가능 항목:**
  - 1회 주문 비용 S ($10~$200, 기본값 $50)
  - 연간 보관 비용률 H (5~50%, 기본값 20%)
- **Tier별 SS & TC 차트:** 서비스 수준별 안전재고 및 연간 총비용 막대 그래프
- **개별 제품 EOQ 상세:**
  - 최적 주문량, 연간 주문 횟수, 연간 총비용, 현재 보유 수량
  - **비용 곡선 차트:** 주문량에 따른 주문비용/보관비용/총비용 변화 (녹색 점선 = 최적점)

#### 탭 4: 알고리즘 비교
- **비교 대상:** K-Means (K=2) vs DBSCAN vs GMM
- **비교 테이블:** 알고리즘별 Silhouette Score, 클러스터 수, Noise 비율, 소프트 할당 여부
- **PCA 2D 시각화:** 3개 알고리즘의 클러스터 결과를 PCA 공간에 나란히 표시
- **GMM 경계 샘플:** 소속 확률 < 0.7인 모호한 제품 목록 (경계 영역 분석)
- **알고리즘 비교 해석 가이드:** 각 지표의 의미와 해석 방법 설명 (expander)

#### 탭 5: S×H 민감도
- **2차원 히트맵:** 주문비용(S)과 보관비용률(H) 조합별 연간 총비용
  - S 값: $25, $50, $75, $100, $150
  - H 값: 10%, 15%, 20%, 25%, 30%
- **Tier별 히트맵:** 각 Tier의 평균 TC를 S×H 그리드에 표시
- **TC 절감 효과 테이블:** 기준(S=$50, H=20%) 대비 최적 조합의 절감액과 절감률

#### 탭 6: t-SNE · UMAP
- **비선형 차원축소:** PCA보다 복잡한 군집 구조를 시각화
- **색상 기준 선택:** 3-Tier 분류 / K-Means (K=2) / ABC 등급
- **t-SNE 시각화:**
  - Perplexity 슬라이더 (5~50, 기본값 30)
  - 작은 값 = 지역 구조 강조, 큰 값 = 전역 구조 강조
- **UMAP 시각화:**
  - n_neighbors=15, min_dist=0.1
  - t-SNE보다 전역 구조 보존에 강점
- UMAP 미설치 시 PCA 시각화로 자동 대체

#### 탭 7: Phase A 회귀 개선
- **목적:** 재고 소진일(DOI) 예측 모델의 정확도를 자동으로 개선한 결과 표시
- **해석 가이드:** 탭 진입 시 expander로 내용 해석 방법 안내
- **모델 개선 과정 및 결과:** 기존 모델 대비 개선된 회귀 모델 성능 비교

#### 탭 8: AI 모델 분석
- **클러스터링 알고리즘 소개:** K-Means, DBSCAN, GMM 등 사용된 알고리즘의 원리와 특성 심층 분석
- **모델 선택 근거:** 각 알고리즘의 장단점 및 본 시스템에서의 적용 이유

#### 탭 9: 용어 사전
- **발주 전략 페이지 전용 용어 설명:** 해당 페이지에서 사용되는 주요 용어와 수식에 대한 상세 설명

## 9. 플로팅 리모컨 TOC & 커스텀 탭 시스템

### 플로팅 리모컨 TOC

**파일:** `utils/styles.py` (`render_mini_toc`, `section_anchor`, `floating-toc-wrapper` CSS)

#### 개요
모든 분석 페이지에서 화면 오른쪽에 고정 표시되는 네비게이션 패널이다. 페이지 내 주요 섹션으로 빠르게 이동할 수 있다.

#### 동작 방식
- **위치:** 화면 오른쪽 중앙에 `position: fixed`로 고정 (z-index: 9999)
- **접기/펼치기:** `<details>/<summary>` 네이티브 HTML 태그로 구현 (JavaScript 불필요)
- **토글 버튼:** 햄버거 아이콘(☰)을 클릭하여 목차 패널을 접거나 펼침
- **섹션 링크:** 각 섹션의 앵커(`section_anchor`)로 이동하는 pill 형태의 링크 목록
- **모드별 TOC 구성:** WMS 시뮬레이터 / 알고리즘 인사이트 모드에 따라 표시되는 섹션 목록이 달라짐
- **호버 효과:** 링크에 마우스를 올리면 녹색 강조 + 좌측 패딩 애니메이션

#### 각 페이지의 TOC 섹션 예시 (대시보드)

| 모드 | 섹션 목록 |
|------|----------|
| 알고리즘 인사이트 | KPI 요약, 제품 그룹 분석, 예측 시뮬레이터, 위험 제품 목록, 분포 차트 |
| WMS 시뮬레이터 | KPI 요약, 재고 상태 분포, 카테고리별 재고 |

### 커스텀 탭 시스템

**파일:** `utils/styles.py` (`render_custom_tabs`)

#### 개요
Streamlit 기본 탭(`st.tabs`) 대신 세션 상태 기반의 커스텀 탭 컴포넌트를 사용한다. 발주 전략 페이지의 알고리즘 인사이트 모드에서 9개 탭을 관리하는 데 활용된다.

#### 동작 방식
- **세션 상태 기반:** `st.session_state[state_key]`에 활성 탭 인덱스를 저장하여 페이지 새로고침 시에도 탭 상태 유지
- **탭 렌더링:** 각 탭 이름을 버튼으로 렌더링하고, 활성 탭에 `primary` 스타일 적용
- **탭별 미니 TOC:** 활성 탭이 변경되면 해당 탭 전용 미니 TOC(`render_mini_toc`)가 플로팅 리모컨에 반영됨
- **탭 목록 (발주 전략):** 발주 긴급도, 제품 그룹·3-Tier, Safety Stock EOQ, 알고리즘 비교, S×H 민감도, t-SNE·UMAP, Phase A 회귀 개선, AI 모델 분석, 용어 사전

## 10. 용어 사전

| 용어 | 영문 | 설명 |
|------|------|------|
| SKU | Stock Keeping Unit | 개별 제품을 구분하는 고유 코드 |
| ABC 등급 | ABC Classification | 매출 기여도에 따른 제품 분류 (A=상위 80%, B=중간 15%, C=하위 5%) |
| 보유 수량 | Quantity On Hand | 현재 창고에 남아 있는 제품 수량 |
| 일평균 판매량 | Avg Daily Sales | 하루에 평균적으로 판매되는 수량 |
| 재고 소진 예상일 | Days of Inventory | 현재 판매 속도로 재고가 며칠 후 바닥나는지 |
| 자동 발주 기준선 | Reorder Point | 재고가 이 수량 이하로 떨어지면 자동 주문을 넣는 기준 |
| 안전 재고 | Safety Stock | 수요 변동이나 배송 지연에 대비해 최소로 유지하는 재고량 |
| 배송 소요일 | Lead Time | 발주 후 물건이 창고에 도착하기까지 걸리는 일수 |
| 유통기한 잔여일 | Days to Expiry | 유통기한까지 남은 날수 |
| 폐기 위험도 | Risk Probability | ML 모델이 예측한 폐기 위험 확률 (0~1) |
| 발주 여유도 | Reorder Urgency | (보유 수량 - 발주 기준선) / 안전 재고. 음수면 긴급 |
| 최적 주문량 | EOQ (Economic Order Quantity) | 주문비용 + 보관비용 합계가 최소가 되는 1회 주문 수량 |
| FIFO | First In First Out | 먼저 입고된 제품을 먼저 출고하는 방식 |
| FEFO | First Expired First Out | 유통기한이 빠른 제품을 먼저 출고하는 방식 |
| 혼동 행렬 | Confusion Matrix | 실제값과 예측값을 교차 비교하는 표. 대각선이 정확한 예측 |
| R² | R-squared | 모델이 데이터를 설명하는 비율 (0~1, 1에 가까울수록 좋음) |
| RMSE | Root Mean Square Error | 예측 오차의 제곱 평균의 제곱근 (낮을수록 정확) |
| MAE | Mean Absolute Error | 예측 오차 절대값의 평균 (낮을수록 정확) |
| 3-Tier 분류 | 3-Tier Classification | Cluster 1 내부 재클러스터링으로 일반/준긴급/긴급 3단계 분류 |
| Sub-clustering | Sub-clustering | 기존 군집 내부를 추가로 세분화하는 클러스터링 |
| DBSCAN | Density-Based Spatial Clustering | 밀도 기반 클러스터링, 노이즈(이상치) 자동 탐지 |
| GMM | Gaussian Mixture Model | 확률 기반 소프트 클러스터링, 각 점의 소속 확률 제공 |
| Safety Stock | Safety Stock | 수요 변동·배송 지연에 대비한 안전재고. SS = Z × σ_d × √LT |
| 서비스 수준 | Service Level | 결품 없이 수요 충족할 확률 (90%, 95%, 99% 등) |
| Silhouette Score | Silhouette Score | 군집 내 응집도 vs 군집 간 분리도 (-1~+1, 높을수록 좋음) |
| Hopkins Statistic | Hopkins Statistic | 데이터에 군집 구조가 존재하는지 검정 (0.75+ = 강한 군집) |
| t-SNE | t-distributed Stochastic Neighbor Embedding | 비선형 차원축소 시각화. Perplexity로 지역/전역 구조 조절 |
| UMAP | Uniform Manifold Approximation and Projection | 비선형 차원축소. t-SNE보다 전역 구조 보존에 강점 |
| TC | Total Cost | 연간 총비용 = 주문비용 + 보관비용. EOQ에서 최소화 대상 |
| PCA | Principal Component Analysis | 선형 차원축소. 데이터의 주요 분산 방향으로 축소 |
