## 1. 핵심 요약 — 로케일 문제점

이 데이터셋은 **인도네시아 로케일** 숫자 표기법을 따른다.
한국/미국과 반대 규칙이므로 별도 변환 없이 pandas로 읽으면 **대규모 파싱 오류**가 발생한다.

### 인도네시아 vs 한국/미국 숫자 표기 비교

| 의미         | 한국/미국 표기 | 인도네시아 표기 | CSV 내 실제 값 |
| ------------ | -------------- | --------------- | -------------- |
| 천 단위 구분 | `1,377`        | `1.377`         | `1.377`        |
| 소수점       | `28.57`        | `28,57`         | `28,57`        |
| 금액 (큰 수) | `$2,084.25`    | `$2.084,25`     | `$2.084,25`    |
| 퍼센트       | `85.36%`       | `85,36%`        | `85,36%`       |

### 영향받는 컬럼별 문제 유형

| 문제 유형                         | 심각도    | 영향 컬럼                                                                             | 문제 설명                                          |
| --------------------------------- | --------- | ------------------------------------------------------------------------------------- | -------------------------------------------------- |
| **마침표 천단위 → 소수점 오파싱** | 🔴 치명적 | `Forecast_Next_30d`, `Reorder_Point`                                                  | `'1.377'` → pandas가 `1.377`로 파싱 (정답: `1377`) |
| **쉼표 소수점 → object 타입**     | 🟡 중요   | `Avg_Daily_Sales`, `Days_of_Inventory`, `SKU_Churn_Rate`, `Order_Frequency_per_month` | `'28,57'`을 숫자로 인식 못해 object 타입으로 로드  |
| **혼합 형식 (천단위+소수점)**     | 🔴 치명적 | `SKU_Churn_Rate` (1건), `Total_Inventory_Value_USD`                                   | `'2.142,90'` → 마침표 제거 + 쉼표 변환 필요        |
| **달러 기호 + 로케일**            | 🟡 중요   | `Unit_Cost_USD`, `Last_Purchase_Price_USD`, `Total_Inventory_Value_USD`               | `'$5,81'`, `'$2.084,25'`                           |
| **퍼센트 기호 + 로케일**          | 🟡 중요   | `Supplier_OnTime_Pct`, `Audit_Variance_Pct`, `Demand_Forecast_Accuracy_Pct`           | `'70,68%'`, `'-7,14%'`                             |

## 11. pandas 자동 파싱 오류 상세

`pd.read_csv()`로 파일을 읽을 때 별도 옵션 없이 호출하면 다음과 같은 파싱 오류가 발생한다.

### 11.1 🔴 치명적 오류 — 값 자체가 틀어지는 경우

이 유형은 **pandas가 숫자로 정상 파싱하지만, 해석한 값이 완전히 잘못된** 경우이다.
데이터 분석 결과가 근본적으로 왜곡되므로 반드시 수정해야 한다.

#### Forecast_Next_30d — 276건 오파싱

```
CSV:  '1.377'  → pandas: 1.377 (float64)  → 실제: 1377 (int)
CSV:  '2.740'  → pandas: 2.740 (float64)  → 실제: 2740 (int)
CSV:  '971'    → pandas: 971.0 (float64)  → 실제: 971  (int) ← 이 경우는 정상
```

- pandas가 `1.377`을 "1점377"로 해석하지만, 실제로는 "1천377"
- 1,000 이상 값은 모두 1/1000로 축소됨
- 평균이 332에서 792로 약 2.4배 차이남

#### Reorder_Point — 10건 오파싱

```
CSV:  '1.170'  → pandas: 1.170 (float64)  → 실제: 1170 (int)
CSV:  '225'    → pandas: 225.0 (float64)  → 실제: 225  (int) ← 이 경우는 정상
```

- Forecast_Next_30d와 동일한 원인
- 1,000 이상 값 10건이 모두 1/1000로 축소됨
- 이 오류는 파생변수 `Stock_Gap = Reorder_Point - Quantity_On_Hand`에도 전파됨

### 11.2 🟡 중요 오류 — 숫자로 인식 못하는 경우

이 유형은 pandas가 **object(문자열)로 로드**하므로, 수치 연산이 불가능하다.
변환하지 않으면 모델 학습에 사용할 수 없다.

## 12. 권장 전처리 코드

아래 코드는 모든 로케일 문제를 해결하는 **권장 전처리 파이프라인**이다.

```python
import pandas as pd

DATA_PATH = 'Inventory Management E-Grocery - InventoryData.csv'

# 1단계: 기본 로드
df = pd.read_csv(DATA_PATH)

# ============================================================
# 2단계: 쉼표 소수점 컬럼 변환
# 규칙: 마침표(천단위) 제거 → 쉼표 → 마침표 → float
# ============================================================
comma_decimal_cols = [
    'Avg_Daily_Sales', 'Days_of_Inventory',
    'SKU_Churn_Rate', 'Order_Frequency_per_month'
]
for col in comma_decimal_cols:
    df[col] = (df[col].astype(str)
               .str.replace('.', '', regex=False)     # 천단위 마침표 제거
               .str.replace(',', '.', regex=False)    # 쉼표 소수점 → 마침표
               .astype(float))

# ============================================================
# 3단계: 마침표 천단위 정수 컬럼 (치명적 오파싱 수정)
# pandas가 float64로 잘못 파싱한 컬럼 → 원본 문자열 재로드
# ============================================================
dot_thousand_cols = ['Forecast_Next_30d', 'Reorder_Point']
df_raw_str = pd.read_csv(DATA_PATH, usecols=dot_thousand_cols, dtype=str)
for col in dot_thousand_cols:
    df[col] = (df_raw_str[col]
               .str.replace('.', '', regex=False)     # 천단위 마침표 제거
               .astype(int))

# ============================================================
# 4단계: 금액 컬럼 변환
# 규칙: $ 제거 → 마침표(천단위) 제거 → 쉼표 → 마침표 → float
# ============================================================
money_cols = [
    'Unit_Cost_USD', 'Last_Purchase_Price_USD',
    'Total_Inventory_Value_USD'
]
for col in money_cols:
    df[col] = (df[col].astype(str)
               .str.replace('$', '', regex=False)     # 달러 기호 제거
               .str.replace('.', '', regex=False)     # 천단위 마침표 제거
               .str.replace(',', '.', regex=False)    # 쉼표 소수점 → 마침표
               .astype(float))

# ============================================================
# 5단계: 퍼센트 컬럼 변환
# 규칙: % 제거 → 쉼표 → 마침표 → float
# ============================================================
pct_cols = [
    'Supplier_OnTime_Pct', 'Audit_Variance_Pct',
    'Demand_Forecast_Accuracy_Pct'
]
for col in pct_cols:
    df[col] = (df[col].astype(str)
               .str.replace('%', '', regex=False)     # 퍼센트 기호 제거
               .str.replace(',', '.', regex=False)    # 쉼표 소수점 → 마침표
               .astype(float))

# ============================================================
# 6단계: 날짜 컬럼 파싱
# ============================================================
date_cols = [
    'Received_Date', 'Last_Purchase_Date',
    'Expiry_Date', 'Audit_Date'
]
for col in date_cols:
    df[col] = pd.to_datetime(df[col])
```

### 13.1 SKU_Churn_Rate 극단 이상치

- 분석 시 이상치 처리 방안 검토 필요 (캡핑, 제거, 또는 그대로 유지)

### 13.2 Inventory_Status 극심한 클래스 불균형

- Out of Stock이 2건(0.2%)에 불과 → 분류 모델 학습 시 문제 발생
- **권장:** Out of Stock → Low Stock에 병합하여 3클래스로 축소

- 1,000건 중 834건(83.4%)이 결측
